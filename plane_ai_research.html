01

The Open-Source Thesis — Why Plane Is Structurally Different

Every other PM tool we've researched (Jira, Linear, ClickUp, monday, Notion, Asana) is closed-source SaaS where the vendor decides which models the AI runs on, where your data lives, and whether you can self-host. Plane is the inverse on all three axes — and that's what shapes its entire AI strategy.

Axis 1 · code

Open-source PM core

Plane is open-source on GitHub at github.com/makeplane/plane. v1.3.1 shipped May 14, 2026. The MCP server is also open source. Closest in spirit to Linear's developer-platform play, but with the actual code public.

Axis 2 · deployment

Self-hostable + air-gapped

"Cloud, self-hosted and air-gapped ready." Docker + Kubernetes with Helm charts. BYO Postgres, Redis, S3-compatible storage. Prime CLI for install / upgrade / backup / monitor. The only PM tool in this research that lists "air-gapped" as a first-class deployment target — appeals to regulated industries and gov.

Axis 3 · AI keys

Bring-your-own-keys (BYOK)

"Bring-your-own keys (OpenAI or Anthropic)." Self-hosted customers pay their AI provider directly. No vendor markup on tokens. Cloud has its own AI bundled in plans, but the model choice is still configurable. Implicit: no Plane-controlled multi-model gateway like ClickUp/Notion/monday.

Why this matters for the AI conversation: Plane is the only PM tool whose AI you can run entirely on your own infrastructure, with your own model keys, with zero prompt-storage on Plane's side. For regulated industries (finance, health, gov, defence) this isn't a feature — it's the entry ticket. Customers cited on plane.so include Aramco, Sony, Dolby, Accenture, Amazon, Zerodha, Stark Bank, Mirador Therapeutics, Government of Lithuania, République Française. That's not a SMB customer profile — it's an enterprise/regulated/government one, and OSS + air-gapped is why.

The Three Modes: Ask / Build / Assign

Plane AI's product surface clusters into three named modes rather than a sprawling brand family like ClickUp (Brain² + Super Agents + Ambient + MAX) or monday (Sidekick + Vibe + Agents + Workflows + MCP + Gateway + Agentalent). Plane's framing is intentionally minimalist.

Mode 1 · query

Ask to know

"Natural-language queries against live workspace data." Ask about status, blockers, doc changes, who's working on what. Web search added Feb 17, 2026 for "real-time external context."

Mode 2 · generate

Talk to build

"Generates project structure, work items, and cycles from a prompt." Plane AI converts a goal description into full project scaffolding — initiatives, projects, epics, cycles, work items with draft briefs.

Mode 3 · delegate

Assign to ship

"@mention an agent on a work item to triage and progress it." Agents act as first-class workspace participants — they appear as assignees, run via Agent Run lifecycle, and are tracked like humans.

"The age of AI does not need more agents. It needs a place for agents to work." — plane.so/ai positioning statement

The cross-cutting layer: Plane AI is exposed externally through the native open-source MCP server (55+ tools, HTTP / PAT / Stdio transports). Internally, the same AI surfaces through Slack and Microsoft Teams via @Plane intake. There is no separate "agent marketplace" like monday's Agentalent — agents are built with Plane's Agent Development Kit and shipped to a Marketplace or kept internal.

Agents — Development Kit, Run Lifecycle, Marketplace

Plane's agent platform is the smallest in scope of any competitor we've researched but has all four elements that matter — identity, lifecycle, surfaces, distribution. The Agent Development Kit + open-source MCP server is the developer story.

Identity
Agents are first-class workspace participants — appear as work-item assignees alongside humans.
Triggers (surfaces)
@mentions in comments, docs, threads (Slack, MS Teams). Direct work-item assignment.
Agent Run
Lifecycle tracking primitive — every agent invocation is an Agent Run, tracked and audited.
Agent Development Kit
Framework to build custom agents. Bundled with the open-source MCP server.
Distribution
Ship to Plane's Marketplace OR keep internal. No public marketplace count yet (vs monday's 17-at-launch Agentalent).
Connectors
First-party: Slack, GitHub, GitLab, Sentry, others. Agent can reach across these via the SDK.

Comparison to Linear's Agent Interaction SDK: Conceptually similar — both make agents first-class users with their own identity, both ship an SDK + open-source reference. Linear's is more polished (8 months of iteration, 18+ external coding agents integrated). Plane's is smaller but has the OSS advantage — you can self-host the entire agent stack.

Plane Compose — Projects-as-Code (YAML)

The Plane feature with no equivalent anywhere else

"Every setting versioned, reviewed, and deployed from your terminal."

Plane Compose lets teams define project structure — initiatives, projects, cycles, work-item templates, labels, states, workflows, dashboards — as YAML manifests checked into Git. Run plane apply to deploy. PR-review your project changes. Roll back via Git. This is GitOps for PM tooling.

What this enables

  • Bootstrap a new team workspace from a template repo — instead of clicking through UI for hours
  • Audit changes to project structure via git blame / git log
  • Multi-environment workflows — dev / staging / prod workspaces, all defined in code, deployed identically
  • Coding-agent compatibility — Plane Compose is the format a coding agent would write to programmatically
  • Compliance-friendly — every change to PM config has an immutable git history

Strategic implication: Plane Compose is the cleanest mechanism we've seen for letting AI agents safely modify project structure at scale. Where ClickUp/monday agents can update settings directly through the API (and bypass review), a Plane agent can write a YAML diff that lands as a git commit — full review, full rollback, full audit. This is the most engineering-team-friendly AI-agent integration in any of the 7 PM tools.

Strategic Vector — "Steerable Systems"

Vamsi Kurama · Co-founder & CEO · 2026 vision

"Steerable systems — humans and agents operate inside the same loop."

Where ClickUp pitches "Converged AI Workspace," monday pitches "AI Work Platform," Notion pitches "Context, collaboration, and action — all in one place," and Linear pitches "Issue tracking is dead," Plane's CEO frames the company's bet around steerability — the idea that org-level decision making (not strategic, but operational mid-level decisions made hourly) is the actual ground where AI changes work.

"Everyone in productivity and project management is talking about consolidation right now — bringing features together and calling it 'All-in-one.' Consolidation without connection doesn't work. Organizations don't just need consolidated tools. They need steerable systems — a steerable system is one where humans and agents operate inside the same loop, where a change in intent propagates through workflows, decisions, and outcomes without friction." — Vamsi Kurama, 2026 vision posts
"This happens a thousand times a week inside every enterprise. Not in the boardroom but below it. The strategic calls are made once a quarter. The decisions that actually move the business get made every hour, by managers doing their best with whatever data they can grab in time. Most of the time, they're flying on instinct and hoping the dashboard catches up." — Vamsi Kurama on operational mid-level decisions as the AI target

Activation & Setup — Cloud vs Self-Hosted

Setup verdict: Cloud is the simplest of all 7 competitors — Plane AI ships on every plan with no Rovo-style toggles, no AI add-on negotiation. Self-hosted is the most powerful but also the most operationally involved — Plane is the only competitor where activation includes "deploy a Kubernetes cluster."

Cloud path

1

Sign up for Plane cloud

Pick Pro / Business / Enterprise. Plane AI is included. No separate AI add-on. 14-day Business trial available.

2

Use Plane AI immediately

Cmd/Ctrl K → Plane AI panel. Try Ask mode ("what's the status of project X?") or Build mode ("create a Q3 launch project"). Web search included as of Feb 17, 2026.

3

Connect Slack / Teams

Settings → Integrations → Slack (or MS Teams when GA). @Plane in any channel to create work items conversationally. Bidirectional thread sync.

4

Connect Plane MCP

Settings → Integrations → Add more → enter https://mcp.plane.so/http/mcp → click Connect → OAuth flow → done. Then add the same URL in Cursor / Claude / VSCode.

5

(Optional) Plane Compose

Define project structure as YAML in a git repo. plane apply deploys. Use for multi-environment, audit-trail, GitOps-style project management.

Self-hosted path

1

Prerequisites

Docker host OR Kubernetes cluster. BYO Postgres + Redis + S3-compatible storage. Anthropic or OpenAI API key.

2

Install via Prime CLI

Single-command install. Configures app, db, queues, storage. Helm charts available for K8s.

3

God Mode setup

Admin panel → configure SMTP, auth (SSO/SAML/LDAP), security, telemetry, AI provider key.

4

Wire Plane AI to your model

BYO Anthropic OR OpenAI key (configured in God Mode). All Plane AI surfaces (Ask/Build/Assign) route through your model.

5

Air-gap (optional)

Restrict outbound network access to only your AI provider endpoint. Plane never sees prompts after this point.
