01

Linear's AI Vision: Context → Execution

Issue Tracking is Dead

Karri Saarinen, CEO (March 24, 2026): "Issue tracking was built for a handoff model of software development... Linear is becoming: a shared product system that turns context into execution."

Linear's thesis: Traditional issue tracking creates overhead and process. Agents compress this by absorbing procedural work. People spend more time on intent, judgment, and taste — less time managing mechanics.

Evidence of agent adoption:

  • Coding agents installed in 75%+ of Linear enterprise workspaces
  • In last 3 months: volume of work completed by agents grew 5x
  • Agents authored nearly 25% of new issues in those workspaces
  • Example: Ramp's internal coding agent now writes 60%+ of merged PRs
  • Example: Coinbase deleted IDEs for 2 weeks — all work via agents in Linear
System Design Shift

From Handoffs to Context

Old model: PM → scope → hand off → Eng → implement → hand off → QA. Each handoff loses context.

New model: Shared product context (feedback, decisions, plans, code) → humans + agents work from same system → agents handle procedural work → execution moves faster.

Who's building this

Linear as Platform

Linear is the "shared product system" that holds:

  • Customer feedback (Linear Asks, Customer Requests)
  • Strategic intent (projects, initiatives, milestones)
  • Execution plans (issues, cycles, dependencies)
  • Implementation evidence (code, PRs, releases)

Agents read all of it. Humans decide what to build; agents build it.

Linear Agent — The Core AI Product

Status: Public beta, March 2026 · Included on all plans (base chat) · Automations/Code Intelligence on Business+ · Free during beta.

What it is: A conversational AI teammate that lives inside Linear. It understands your workspace context (issues, projects, history, decisions) and can take action on your behalf. Available:

  • In dedicated chat: ⌘/Ctrl + J from app
  • Inline in comments: @Linear mention
  • In Slack: @Linear in any channel
  • In Microsoft Teams: @Linear in Teams channels
Chat surface

Dedicated Agent Chat

Main conversational interface. Persistent chat history grouped by recency. Can open multiple tabs for multi-tasking. Shows agent reasoning in real time as it works.

Example prompts:
"Summarize project progress with risks and next steps"
"Which issues are at risk of rolling over?"
"Draft a stakeholder update on outcomes"

Inline surface

@Linear in Comments

When drafting a comment, mention @Linear to co-author. Useful for status updates, summaries, decision recaps — results live on issue for team visibility.

Example: Writing an issue comment, ask Linear to "Turn these notes into a weekly update"

  • Personal: Only you can use them (default)
  • Repeatable: Invoke with slash command or auto-trigger
  • Focused: One job well. Can have multiple steps, but shouldn't pack too much.
  • Refineable: Ask Linear to optimize — remove unnecessary steps

Automations — Trigger Agents on Triage

Status: Public beta, available on Business+ plans

Run agent workflows automatically when an issue enters triage. Define open-ended instructions:

  • Agent reads the incoming issue
  • Applies your automation instruction
  • Takes action (add summary, classify, attach docs, etc.)
  • No human prompt needed

Example automation: "If this issue mentions 'SSO', attach our SSO troubleshooting guide and assign to the Security team."

Configure in: Team Settings → Triage → Agent behavior. Scope to specific issue types or apply workspace-wide.

Guidance — Shape Agent Behavior

Set instructions at three levels:

Workspace Guidance

Applies to everyone. Admin controls who can edit. Examples: "Keep all summaries to 100 words", "Use [Style Guide]", "Always mention customer impact".

Team Guidance

Team-specific instructions. Reference in workspace guidance. Useful for teams with unique processes.

Personal Guidance

Individual preferences (Settings → Agent personalization). Overrides workspace guidance for that user.

04

Triage Intelligence — Automated Issue Processing

Status: Public beta, September 2025 · Business+ plans · Technology preview

What it solves: Organizing a backlog is manual work that doesn't scale. Product teams struggle with: duplicate detection, missing properties, inconsistent classification. Triage Intelligence automates this.

Architecture: Better Search + Agentic Reasoning

The Problem: Small Models Couldn't Handle Nuance

Initial approach used small models (GPT-4o mini, Gemini 2.0 Flash) with rigid prompts. They worked for straightforward cases but failed on context-sensitive decisions. If something important was missing, they had no way to retrieve it.

The Solution: Agentic Architecture with Frontier Models

Shift to larger models: GPT-5 + Gemini 2.5 Pro. These can handle complex reasoning AND dynamically pull context.

How it works:

  • New issue enters triage
  • Search engine finds candidate related issues (via vector similarity)
  • Agent model reasons: duplicate? related? new?
  • If uncertain, agent pulls additional context from workspace
  • Model evaluates, suggests: links, labels, assignee
  • Explains reasoning in plain language

Triage Intelligence Output

Detection

Duplicate Detection

Identifies when incoming issue is likely duplicate of existing work.

Linking

Related Issues

Suggests related issues in backlog. Helps see connected work at a glance.

Classification

Property Suggestions

Recommends labels, assignee, project based on similar issues in history.

Design for Trust & Transparency

Larger models are slower than Linear's usual deterministic systems. Linear designed UI to make that acceptable:

  • Visible reasoning: Hover over suggestion → see full explanation + alternatives
  • Thinking state: Shows progress indicator + timer so system doesn't feel idle
  • Thinking panel: Deep dive view showing all context the model pulled in, decisions made, how guidance shaped outcome
  • Additional Guidance: Fine-tune at workspace/team level with plain language prompts. Steer suggestions toward your priorities.

Principle: "Build trust, keep reasoning transparent, make it feel like part of Linear." — Linear team's Agent Interaction Guidelines

Code Intelligence — Codebase Awareness

Status: Public beta, May 14, 2026 · Business+ plans · Free during beta

What it is: Gives Linear Agent controlled read access to your codebase. Agent can understand implementation details, architecture, recent changes — without digging through GitHub.

What Linear Agent Can Do with Code Intelligence

  • Answer technical questions: "How does the payments service handle failed transactions?"
  • Non-technical → technical: PMs ask "How is this feature implemented?" without finding an engineer
  • Root cause analysis: "What changed in the auth flow recently? Could it cause this bug?"
  • Spec writing: PMs write sharper specs informed by current implementation
  • Support/Sales: Answer customer questions about product behavior with more confidence

Setup

1

Install GitHub integration

Workspace admin enables GitHub integration in Linear settings. Authorize Linear to read repository content.

2

Enable Code Intelligence

Go to Settings → AI → Code Intelligence. Turn it on.

3

Choose repositories

Admin selects which repos agent can access. Choose whether all workspace members can query or only those with GitHub access.

Scope: Code Intelligence respects existing GitHub permissions. If a workspace admin restricts to "only users with GitHub access", the agent only gives code answers to those users.

Linear Asks — Feedback Collection

Status: General Availability · Enterprise feature (web forms) · Slack/email always available

What it is: System to capture internal and external requests. Convert unstructured input (Slack, email, web form) into structured Linear issues.

Input Channels

Slack / Teams integration

Mention @Linear in Slack → "file a bug" or "create a feature request". Linear creates issue in triage.

Email capture

Forward emails to Linear inbox. Auto-convert to issue. Submitter stays in email loop.

Custom web forms

Enterprise only. Teams create Asks page with custom forms (bug reports, feature requests, HR, IT tasks). Anyone in company can submit without Linear account.

Key feature: Triage Intelligence routing

When Asks are submitted, they go into triage where Triage Intelligence can:

  • Detect duplicates automatically
  • Suggest the right team
  • Apply labels + properties
  • Link related work

Admin Setup

1

Enable MCP in workspace

Settings → AI & Agents → Enable MCP servers toggle

2

Approve allowed servers

Admin chooses which MCP servers users can connect to. Restrict to approved list or allow custom servers.

3

Users connect their accounts

Settings → Agent personalization → Add server. Complete OAuth if needed.

Permission model: Agent can only call MCP servers that the user has connected. If you haven't authenticated with Notion, Linear Agent can't access your Notion workspace — even if workspace admin enabled Notion.

Summary: Linear's AI Architecture

Core Thesis

Issue tracking created overhead. Agents + rich context can eliminate that overhead. Linear = the shared product system where context lives + agents execute.

Three Pillars of AI

  • 1. Linear Agent: Conversational interface. Read workspace, take action, learn skills.
  • 2. Triage Intelligence: Automatic issue organization. Detect duplicates, link related, classify.
  • 3. Code Intelligence: Codebase awareness. Agents understand implementation, architecture, changes.

Execution Vectors

  • MCP Integration: Bring external context (Slack, GitHub, Notion, Datadog, etc.) into Linear workflows
  • Coding Agent Partnerships: Deep links + context hand-off to Cursor, Codex, GitHub Copilot
  • Automations at Triage: Agent workflows trigger on issue arrival, not manual prompts
  • Releases Tracking: Know what's actually live to customers (not just merged)

Where Linear Differs from Competitors

  • Agent is first-class UI element, not an addon in sidebar
  • Deep code access (Code Intelligence) — not just issue text
  • Emphasis on "self-driving" workflows — agents move work forward autonomously
  • MCP ecosystem for any external tool (not pre-built integrations only)
  • Speed + simplicity over feature richness
