01

The Three Pillars

Linear's AI surface is intentionally narrow and does not match the breadth of Jira's Rovo or ClickUp's Brain. It clusters into three coherent product pillars — and one cross-cutting platform layer for external agents. Linear deliberately avoids generic chat, content generation, meeting AI, or "ask anything" features; their bet is depth in a narrow developer-product workflow.

Pillar 1 · inline

Triage Intelligence

Self-driving triage intake. Suggests (and optionally auto-applies) assignee, team, project, labels, related issues, and duplicate links on every new issue. Originally launched as "Product Intelligence" (Aug 2025); renamed Sept 2025. Business+

Pillar 2 · everywhere

Linear Agent

The native AI teammate. Chat (⌘J), inline @Linear in any text field, Slack/Teams intake, mobile, agent toolbar. Composes issues from conversations, drafts updates, executes Skills, runs Automations. Public beta Mar 24, 2026. All plans (chat + skills)

Pillar 3 · platform

Agent Platform

Linear-as-substrate for any AI agent. Agent Interaction SDK + Linear MCP server + OAuth actor=app. External coding agents (Cursor, Codex, Devin, Copilot, Warp, Amp, v0…) install as first-class non-billable users with delegate semantics.

Cross-cutting layer: Code Intelligence (public beta May 14, 2026) — gives Linear Agent controlled read access to your repositories. Unifies code context and product context in one workspace, so non-engineers can ask "how does X work" without tracking down an engineer. Business+

02

Linear AI = Semantic Search + Frontier LLMs + Workspace Context

Unlike Rovo's RAG-over-everything pitch, Linear is explicit that the value is in the structured context the platform already holds — issues, projects, initiatives, customers, feedback, code (via Code Intelligence), and external systems via MCP. Linear's engineering blog confirms an agentic architecture on frontier models, not a small-model RAG pipeline.

Linear AI= Hybrid semantic search// rebuilt 2025, unified across product
+ Agentic LLM layer// GPT-5 + Gemini 2.5 Pro (per eng blog, Sep 2025)
+ Structured workspace context// issues, projects, initiatives, customers, feedback
+ Code Intelligence (beta)// indexed codebase access for the Agent
+ External MCP servers// Granola, Glean, Notion, PostHog, your own
+ "No training on your data"// explicit privacy positioning

Key architectural choice: Linear's first Triage Intelligence build used "smaller models with tightly scoped prompts and rigid workflows" (GPT-4o mini, Gemini 2.0 Flash). They moved to GPT-5 + Gemini 2.5 Pro in an agentic loop so the model can pull additional workspace context on demand instead of being prompt-stuffed up front. No accuracy benchmarks are published — Linear leans on UX (live "thinking" trace, reasoning hover) for trust instead of numbers.

Latency tradeoff (named explicitly): "If something takes more than a few hundred milliseconds, we try to make it faster… frontier reasoning models are fast compared to a human, but slower than Linear's usual deterministic systems." This is why every AI surface has a visible thinking-state indicator with a timer — a Linear-Method-consistent way of saying "yes, AI is here, and it's slower, deal with it on purpose."

Agent Anatomy — Sessions, Activities, Delegation

Linear's agent platform is the most thought-out in the PM-tool space. The model is: agents are first-class users with their own identity, OAuth token, and scoped permissions, but they are delegated to rather than assigned, preserving human accountability.

"Issues can only be assigned to humans, and only delegated to agents. Every agent-involved issue retains a human assignee responsible for the outcome alongside the delegated agent doing the work." — Our approach to building the Agent Interaction SDK, Linear, Aug 2025
Identity
OAuth actor=app mode → dedicated app-user, not billable. Workspace-scoped install (per-workspace app ID). Replaces older actor=application.
Scopes
app:assignable · app:mentionable · customer:read|write · initiative:read|write. Apps using actor=app cannot also request admin scope.
Agent Session
Lifecycle primitive. Auto-created on @mention or delegation. States: waiting for input, actively working, completed, errored. Sessions persist with issue and restore on un-archive.
Agent Activity
Structured items inside a session: thoughts, tool calls, elicitations, final responses. Linear renders the UI automatically — developers don't paint the surface.
Delegation model
Issue retains human assignee. Agent is added as delegate. fromDelegate / toDelegate fields appear in IssueHistory.
Webhooks
"Agent session events" category. First webhook is AgentSessionEvent created with promptContext (issue, comments, guidance). Agent must emit a thought activity within 10 s to acknowledge.

OAuth + GraphQL — installation snippet

// 1. Authorize as app, not as user
//    actor=app + admin permission required for install
//    https://linear.app/oauth/authorize?actor=app&scope=app:assignable+app:mentionable+...

// 2. Fetch app's workspace-scoped identity
query Me {
  viewer { id }
}

// 3. Subscribe to "Agent session events" webhooks
//    Receive AgentSessionEvent { type: "created", agentSession, promptContext }
//    Reply with createAgentActivity { type: "thought", ... } within 10s.

Agent triggers — surfaces an agent appears on

Direct @mention

Comments, document editor, issue description. Creates a session with the mention as promptContext. Mention in description triggers a session (added Sep 2025).

Issue delegation

Assigning to an agent moves them to the delegate slot. Agent receives session with full issue context. app:assignable scope required.

Automation rule

"Run agent X when issue enters triage / changes state." Configured per team. Business+ only.

External chat (Slack, Teams, Intercom, Zendesk, Gong)

@Linear in conversation → agent reads thread context, files/edits issues. Natural-language confirmations recognised even without explicit @Linear mention in approval flows (May 2026).

Code Intelligence — Code as Shared Product Context

Public beta · May 14, 2026 · free during beta

"Repositories as shared product context your whole team can use"

Code Intelligence gives Linear Agent controlled access to your codebase. Once enabled (Business+ / Enterprise), the agent can answer code-aware questions — explain how a feature works, investigate likely causes for a bug, draft technical specs grounded in actual source, and tell non-engineers who owns a system and what recently changed without an engineer in the loop.

Coupling: Code Intelligence is the substrate for Linear's own coming Coding Agent — the proprietary one Saarinen previewed in "Issue tracking is dead." It writes code, fixes bugs, and presents code diffs natively in Linear (Linear Code Reviews entered private beta Jan 22, 2026).

Why it matters strategically: Code Intelligence collapses the line between "issue tracker" and "engineering knowledge graph." It's the technical mechanism behind Saarinen's "context and agents" thesis — and it's the move that puts Linear in direct competition with both Atlassian's DX acquisition (engineering intelligence) and Notion AI's codebase Q&A.

Linear MCP — Bidirectional Context Layer

Linear MCP is bidirectional — and that's the strategic point

Outbound: Linear ships an MCP server that exposes Linear's read/write surface as tools. Any MCP-capable client (Cursor, Claude.ai, ChatGPT, v0, Figma Make, Notion Custom Agents, Gemini Enterprise) plugs into Linear and operates against your workspace with your permissions.

Inbound: Since April 23, 2026, Linear Agent can connect to external MCP servers (Granola, Glean, Notion, PostHog, or your own). Admins control access via allowlists and workspace-level MCP permissions.

Endpoint: /mcp (current) — /sse deprecated. Auth: OAuth or personal API key. Tools annotated with deprecation/parameter metadata.

Linear MCP tools (selected — exposed to external clients)

save_issue (create+update combined Feb 2026) save_project save_document (create+update; accepts initiative, cycle) list_issues (paginated, filterable) list_comments (paginated) list_users (filter by team) list_issue_labels (paginated) list_cycles semantic_search (multi-type, Sep 2025) manage initiative + milestones + updates project labels CRUD project state & members issue relations (blocking/related/duplicate) add/remove project from initiative add/remove issue from milestone Slack/Teams channel link fields SLA status on issue responses "unknown tool parameters" validation (May 2026)

Pattern: convergent save_* naming. Linear deliberately collapsed create_issue + update_issue into save_issue in Feb 2026, then did the same for documents. Reasoning is implicit in the changelog: cleaner LLM tool-call semantics — fewer tools, less ambiguity, less token weight in tool descriptions.

How Linear MCP differs from Atlassian's Rovo MCP

Linear MCP

  • Bidirectional — Linear Agent uses external MCPs too
  • Auth = standard OAuth, no AI-side toggle
  • Single endpoint /mcp, public & discoverable
  • Available on all paid plans
  • Acts as the user when called from external clients

Atlassian Rovo MCP

  • Outbound only (Claude → Jira). Rovo agents can use external MCP in EAP, but gated.
  • Auth requires "AI enabled apps" toggle per product + Rovo activation
  • 7 interactive tools (smaller surface)
  • Available on Standard+ but with the activation headache documented in the Jira research
  • Also acts as the user

Agent Interaction SDK + AIG

Two artifacts shipped July 30, 2025: the Agent Interaction SDK (developer preview, TypeScript) and the Agent Interaction Guidelines (AIG) — a "living document" of design principles for agent UX. Together they are Linear's bet that a thoughtful platform API will out-compete Jira's Studio-style no-code IDE for the agent developer market.

AIG — design principles (quoted verbatim)

Platform-native

"An agent should inhabit the platform natively." No separate page, no sidebar chat-with-the-vendor. Agents appear in the same surfaces humans use — issues, comments, documents.

Disclosure

"Agents should always disclose that they are agents." Agent users render with distinct identity badges.

Instant feedback

"An agent should provide instant feedback." Hence the <10 s thought-acknowledgement requirement and the live thinking-state UI.

Transparency of state

"An agent should be clear and transparent about its internal state." Activities are structured: thoughts, tool calls, elicitations — not opaque text blobs.

Human accountability

"An agent cannot be held accountable." → delegation, not assignment. Humans stay on the hook for outcomes.

Flexible guardrails

"Flexible guardrails" — opt-in scopes, opt-in surfaces. Platform shapes behaviour without restricting it.

Strategic implication: Linear is explicitly not trying to be MCP. AIG codifies UX behaviour above the protocol layer — the parts MCP doesn't define (status, disclosure, accountability, delegation). Linear positions itself as the substrate where many agents (including its own) behave consistently. Compare to Atlassian, which extended Forge into a no-code agent IDE (Rovo Studio) — a different bet on lowering the build floor rather than standardising agent behaviour.

Reference TypeScript implementation

Linear ships linear/weather-bot — a sample agent on the TypeScript SDK + Cloudflare Workers — as the canonical starting point. Webhooks: AgentSessionEventWebhook, AgentActivityWebhookPayload. New issueRepositorySuggestions query exposes Linear's confidence-ranked repo guess for an agent session (Jan 2026).

Pricing & the Autonomy Spectrum

Linear deliberately bundles AI into seats rather than monetising it as a separate add-on. Karri's "Self-driving SaaS" essay frames this on a three-level autonomy spectrum, and the pricing aligns with it: assistive AI is free, moderate-autonomy AI is Business-tier, full-autonomy ("agent does everything") is reserved for compute-heavy capabilities that may shift to usage-based pricing.

L1 · Assistive

Free / all plans

Linear Agent chat surface · Skills · @Linear in Slack & Teams · Semantic search · AI filter · Linear Asks intake · Linear MCP server.

L2 · Moderate

Business+ (~$10–16/user/mo)

Triage Intelligence + auto-apply · Automations · Pulse · Customer-conversation intake (Intercom/Zendesk/Gong) · Linear Asks AI · Code Intelligence (beta-free, then Business+).

L3 · Full

Enterprise / future usage-based

Linear Code Reviews (private beta) · Linear Coding Agent (preview/coming) · High-volume Automations + Code Intelligence may shift to usage-based pricing beyond a threshold (advance notice promised).

Data philosophy (quoted from linear.app/ai): "Your information is never used to train AI models and remains secure at every step." This is a positioning lever vs Atlassian (who train on customer data with opt-out, depending on plan) and the open-mode AI vendors. Among the seven PM-tool competitors, only Linear and (partially) Notion stake out this claim plainly on a marketing page.

Strategic Vector — "Issue Tracking Is Dead"

Karri Saarinen · Mar 24, 2026 · linear.app/next

"Linear is the shared product system that turns context into execution."

Linear's positioning manifesto, published alongside the Linear Agent public beta. The arc: issue trackers were designed for human handoffs; the next system is designed for humans + agents sharing context. Linear stops calling itself an issue tracker and starts calling itself a "shared product system."

"The next system is not designed around handoffs. It is designed around context and agents."
"Agents are not mind readers. They become useful through context — feedback, ideas, strategy, decisions, and code must live in one shared system."

Next 3 months (May → Aug 2026)

  • Code Intelligence GA
  • Linear Code Reviews public beta with structural diffs for agent output
  • Linear Coding Agent (preview) — proprietary, wraps Devin/Cursor with deeper integration
  • Skills library marketplace signals (likely)

  • Code Diffs view "built for both humans and agents to iterate together"

Next 12 months

The Linear vector summary: "PM layer for agentic engineering." Linear is not trying to be a general PM platform with bolted-on AI like Asana or Monday — and not trying to be a wide marketplace like Jira's Rovo. They are intentionally narrow, deep, and developer-workflow-specific. The bet: when most code is written by agents, the orchestration layer becomes the most valuable software in the engineering org — and Linear wants to own it.

Activation & Setup — How to Actually Turn It On

Versus Jira: Linear's activation story is dramatically simpler. There is no "two switches that must both be ON," no per-product enable, no business-email domain trap, no rollout-by-region delay. The complexity is in configuring AI (auto-apply rules, additional guidance, agent permissions), not in turning it on.

1

Pick the right plan

Free → chat surface, Skills, MCP, AI Filter, semantic search. Business+ ($10–16/user/mo) → Triage Intelligence, Automations, Pulse, customer-conversation intake, Code Intelligence. Enterprise → private-beta features like Code Reviews.

2

Open Linear Agent

⌘/Ctrl + J in the desktop or web app opens the agent overlay. No admin toggle required to use chat on any plan.

3

Configure Triage Intelligence

Settings → Product Intelligence. Set per-team / sub-team Additional Guidance. Decide which suggestion types auto-apply and which only suggest (e.g. always auto-apply team, only auto-apply bug label).

4

Wire intake channels

Linear Asks (Slack/email/web form) · Intercom / Zendesk / Gong / Datadog integrations. Each becomes a Triage Intelligence-routed source.

5

Install coding-agent integrations

Add Cursor / Codex / Copilot / Warp / Devin / Claude Code as workspace agents. Configure per-issue "Open in <tool>" deeplink prompts. Build a Custom Local Script deeplink for unsupported tools.

6

Connect MCP (both directions)

Outbound: in Cursor / Claude / ChatGPT settings, add Linear MCP via /mcp endpoint (OAuth flow). Inbound: Admin Hub → MCP servers → allowlist Granola / Glean / Notion / PostHog / your own.

7

Enable Code Intelligence (Business+)
