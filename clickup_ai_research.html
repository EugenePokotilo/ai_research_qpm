Super Agents™ — Agents as First-Class Users

Super Agents launched December 22, 2025 (initial preview) → fully shipped January 29, 2026 ("Release Notes Takeover"). Built on the Codegen technology acquired December 23, 2025 — a deliberate fast-cycle: acquire on the 23rd, sunset standalone product Jan 16, fully relaunch by Jan 29.

"Super Agents are THE WORLD'S ONLY HUMAN-LEVEL AI AGENTS that work as teammates… They show up just like teammates, because under the hood, they're modeled as real users." — clickup.com/brain/agents

The 7 capabilities of a Super Agent (verbatim from the page)

Memory
"Episodic memory, agent preferences memory, short-term memory and long-term memory." Decisions stored in editable Docs.
Knowledge
Vast databases + learning from every interaction. Real-time syncing via ActionQuery + fine-tuned embeddings.
Collaboration
Works with humans AND other agents — multi-agent delegation under BrainGPT's sub-agent architecture.
Skills
"500+ human skills" — email, scheduling, reporting, image generation, code, sync, audit, etc.
Autonomous
Independent decisions and actions. Reflection loop: "execution loops that ensure Agents constantly reflect on work they're doing."
Ambient
Runs quietly in the background — works on schedules or triggers without explicit human prompts.

How a Super Agent gets triggered (4 surfaces)

@mention anywhere

Tag the agent in a task, doc, comment, or chat. Agent reads context, executes, posts result.

Assign as task owner

Delegate a task to an agent like a human teammate. Agent executes end-to-end and proactively escalates.

Direct message (DM)

Open a 1:1 chat with the agent. Iterate naturally. Edit tone, instructions, tools through the conversation — no Builder revisit needed (improved May 2026).

Schedule / Automation trigger

Daily morning summaries, hourly reminders, event-triggered playbooks. May 2026 added hourly schedules with guardrails against overlapping runs.

Super Agent Builder ("Super Agent Studio")

No-code · natural language

"The world's only infinite agent catalog"

Describe the job in plain English → Builder defines scope, knowledge sources, tool restrictions, behavior. "One prompt spins up an entire team" — Builder can auto-delegate goals to multiple agents at once.

Agent categories shown on /brain/agents: PM Agents · Sales Agents · Coding Agents · Designer Agents · Certified Agents (marketplace). The launch blog names Project Manager Agent as a flagship pre-built.

Permissions & security model

  • Modeled as real users in ClickUp's existing data model — agents have user accounts with explicit role-based access.
  • Default scope: public Workspace data only. Private context shared selectively.
  • Audit everything: every action (post, summarize, create, assign, status change, update DB, generate report) logged with timestamp + location.
  • Real-time audit logs + admin granular permissions.
  • "Zero data retention. Zero training." — page claim: "More secure than using OpenAI, Gemini directly."
  • Reflection loop on every action for self-checking.

BrainGPT — the proprietary engine

Optimized orchestration

"Route to the best models from intent" — picks GPT vs Claude vs Gemini per sub-task. The technical reason ClickUp can ship multi-model without UX chaos.

Sub-Agent architecture

A Super Agent can spawn sub-agents that handle slices of work in parallel and report back. Inherited from Codegen's "autonomous PR" stack.

Deep Research & Compression

"Research optimally from compressed context" — long-running research with context windowing to handle large workspaces without token bloat.

Self-Learning

"Every interaction, job, and piece of human feedback improves Super Agents." Continuous improvement without retraining the underlying foundation model.

"100s more" pre-built variants per /brain/ambient-agents. Connected app integration: "work with data from your Google Drive, GitHub, Salesforce, and 20+ other apps." This is the surface that consumes the Qatalog ActionQuery engine most aggressively.

Enterprise Search — Qatalog ActionQuery Engine

The Nov 2025 Qatalog acquisition was the single biggest infrastructure change to ClickUp AI in 2025–2026. Qatalog's ActionQuery engine — a "permission-aware, no-index" AI search platform formerly used at Nvidia, Twitter, Airbnb — is now embedded in ClickUp.

"Qatalog's ActionQuery AI engine — a permission-aware search technology — will be deeply integrated into ClickUp's platform, enabling users to instantly surface information from any connected app without leaving ClickUp." — Qatalog acquisition announcement, Nov 12, 2025

What ActionQuery does inside ClickUp

100+ connected enterprise apps

Drive, Notion, Slack, Gmail, GitHub, Salesforce, Confluence, Dropbox, MS Teams (logos shown on /brain/agents) — and "20+ other apps" per /brain/ambient-agents.

Permission-aware retrieval

Respects per-document, per-app permissions. User only sees what they can see in the source app. No "AI bypass" of access controls.

No-index architecture

Doesn't store enterprise data permanently in ClickUp. Queries hit source systems live (similar to Atlassian's "Direct" connector type but native).

Real-time 2-way sync

Bidirectional — agents can both read from connected apps and write back via integrations (where the source app supports it).

ClickUp MCP — Official OAuth Server + Community Variants

Native MCP server, public beta, OAuth-only

Endpoint: https://mcp.clickup.com/mcp

Auth: OAuth only — "you cannot authenticate using your own API keys or Auth access tokens." Forces multi-user-aware install.

Admin gate: In Team / Enterprise / Edu workspaces, only owners and admins can configure MCP servers.

Plan availability: All plans (public beta).

Rate limits (without Everything AI): Free: 50 calls / 24 h · Unlimited+: 300 calls / 24 h. Higher with Everything AI add-on.

Safety constraint: "For the time being, ClickUp hasn't added any deletion tools as a safety measure." Read + write + update only — no destructive operations exposed to LLM clients.

Setup snippet (works for Cursor, Claude Desktop, VS Code, ChatGPT)

{
  "mcpServers": {
    "ClickUp": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://mcp.clickup.com/mcp"]
    }
  }
}

What ClickUp MCP exposes

ClickUp's official docs don't enumerate the full tool surface, but example prompts and the community taazkareem fork suggest coverage of:

Tasks (create, update, list, comment) Lists / Folders / Spaces Docs (read, edit) Sprints & cycles Checklists Tags Time tracking Files / attachments Chat messages Custom fields read ⚠ NO deletion tools (safety)

Official vs community MCP — when to use which

ClickUp official MCP (mcp.clickup.com)

  • OAuth-only, multi-user safe
  • Zero install — hosted by ClickUp
  • Curated tool set, no deletions
  • Rate-limited per plan
  • Pick when: enterprise, Cursor/Claude/ChatGPT multi-user, no infra to run anything

Community (taazkareem / nazruden)

  • API key / personal token auth
  • Local execution (npx-installed)
  • Complete API coverage including delete
  • Filter flags to shape read-only or task-focused agents
  • Pick when: solo dev, need full surface, want to script behaviour
